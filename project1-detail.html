全自动鼻镜检测与AI初筛系统 与中山三院合作项目

基于法奥意威 FR5 协作机器人，挂载内窥镜，完成从体外面部定位到体内腔道跟随的完整自主流程。提供 ROS2 分布式节点与纯 Python 单脚本两种实现，核心算法一致，适配不同部署场景。

视觉伺服手眼集成人工势场 ROS2PythonMediaPipe 卡尔曼滤波状态机FR5协作机器人

本项目构建了一套完整的“体外定位—体内导航”视觉伺服系统，机械臂从任意起始位姿出发，首先依靠面部特征将鼻镜末端引导至鼻腔入口，随后进入视野受限的腔道内部，依靠腔内光影变化自主纠偏并抵达目标深度。整个工程兼顾了实时性、鲁棒性与安全性，并在两种架构下得到了验证。

双版本实现： ROS2 版充分利用分布式节点、话题/服务通信与多线程调度，适合多模块协同与数据记录；纯 Python 脚本版剥离框架依赖，单文件即运行，部署极简。两者共享相同的视觉算法、状态机设计与运动控制策略。

第一阶段：体外视觉定位与逼近

摄像头实时画面经裁剪后送入 MediaPipe 人脸关键点模型。系统通过鼻翼两侧的几何插值动态计算出目标鼻孔的亚像素坐标，并利用鼻翼间距作为天然尺度参考，实时估算像素到物理毫米的转换比例，从而摆脱了对固定标定距离的依赖。

运动决策由带迟滞阈值的状态机驱动：只有当像素偏差持续超过“进入阈值”才触发 XY 对齐，低于“退出阈值”且稳定多帧后才切换为 Z 轴进近。状态切换期间采用线性插值混合运动指令，使机械臂轨迹平滑过渡。此外，针对 MediaPipe 偶发的点位跳变，系统内置了异常帧过滤器（位移跳变与宽度跳变双重检测），并结合指数滑动平均滤波，确保视觉信号输出稳定。

在工程实现上，ROS2 版将感知、决策、控制拆分为独立节点，通过话题与动作服务异步交互；纯 Python 版则将所有逻辑封装于单一主循环中，直接调用法奥 SDK 的底层 RPC 接口完成工具坐标系下的偏移移动。二者均包含目标丢失后的等待重试与后退复位机制。

第二阶段：体内受限环境力场导航

工具进入腔口后，视野瞬间变暗且缺乏全局特征。系统首先进入“盲走”阶段：以小步长沿工具 Z 轴开环推进，同时持续监测视野中是否出现稳定的暗区纹理。一旦连续多帧检测到有效目标，即自动切换至视觉闭环控制，解决了入口处的阶段衔接问题。

闭环导航的核心是基于人工势场（APF）的力场计算。图像经 CLAHE 增强与高斯模糊后，通过百分位阈值分割出暗区（引力源）与亮区（斥力源），实时合成引导工具向画面中心靠拢的合力矢量。为提高信号平滑度，系统对力场分量施加 EMA 滤波，并对目标点位置运行二维卡尔曼滤波器进行轨迹预测，有效抑制了腔内图像噪声引起的抖动。

力场矢量通过一个非线性转换器映射为机械臂的动作：力的方向决定工具坐标系的旋转方向（Rx/Ry），力的大小则用于动态调整旋转增益与 XY 平移步长——偏离中心越远，动作越果断；越靠近中心，调整越精细。转换器还包含死区阈值、单次最大角度/平移限幅以及每阶段累积量限制，防止过冲。状态机在“旋转对齐”与“直向前进”之间采用滞回切换，过渡期间通过线性插值实现运动指令的淡入淡出，进一步柔化机械臂轨迹。

在架构层面，ROS2 版将力场转换、卡尔曼滤波、状态机评估分散在节点回调中，并利用多线程执行器实现图像处理与运动控制的并发；纯 Python 版则以紧凑的主循环完成所有计算，并通过线程池异步下发运动指令，同时借助 OpenCV 绘制丰富的调试界面（力场箭头、滤波前后目标点、深度进度条等），使算法内部状态一目了然。

安全与鲁棒性设计

  • 多重滤波链路： EMA 平滑力场与像素距离，卡尔曼滤波预测目标轨迹，异常帧拒绝剔除跳变值，共同保证视觉信号可靠。
  • 滞回状态机 + 帧计数防抖： 避免在阈值边界频繁切换状态，防止机械臂震颤。
  • 累积量安全限制： 单阶段旋转角度上限、总深度上限、平移累积上限，防止运动超调或碰撞。
  • 错误恢复： 目标丢失超时后自动后退重试；ROS2 版利用服务重试机制，纯 Python 版内置 SDK 重发逻辑，应对通信抖动。

技术栈摘要

视觉：OpenCV、MediaPipe、CLAHE、自适应阈值分割。
滤波与控制：指数滑动平均 (EMA)、二维卡尔曼滤波、人工势场 (APF)、非线性增益映射、线性插值过渡。
架构：ROS2 (Humble) 节点/话题/服务/多线程执行器；Python 多线程与线程池；法奥机器人 SDK (RPC / ROS2 服务)。
状态管理：有限状态机 (FSM) 配合迟滞阈值与帧计数防抖。

本项目从体外的特征定位到体内的力场跟随，形成了一套可部署、可扩展的视觉伺服解决方案。ROS2 版展现了分布式机器人系统的工程规范，纯 Python 版则凸显了算法内核的鲁棒性与轻量化集成能力。
